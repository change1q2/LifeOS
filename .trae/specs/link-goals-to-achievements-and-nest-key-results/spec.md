# 目标关联成就与关键结果无限层级 Spec

## Why
当前目标管理支持在创建目标时自动向「学习成长、旅行日记、心情心态、健康习惯、财务管理、社交人脉、收获感悟」等模块生成关联记录并同步进度，但缺少与「成就墙」的关联。同时关键结果虽然数据模型是递归的，但前端表单、展示与后端切换接口只支持到三层，且父节点不会随子节点全部完成而自动完成。为了让目标达成能够自动解锁成就，并支持更灵活的目标拆解，需要补齐这两块能力。

## What Changes
- 在目标数据模型中新增 `linked_achievement_id` 字段，用于关联成就墙中的手动成就（可选）。
- 在目标表单中新增「关联成就」选择器，按成就所属模块与父子层级展示；若选中的成就还有子成就，可继续选择子层级。
- 当目标进度达到 100% 时，自动将关联成就的 `locked` 设为 `false`，并把 `date` 设为当天，实现目标达成即解锁成就。
- 保持并完善「创建目标后自动在关联模块创建记录」的能力，继续支持学习成长、旅行日记、心情心态、健康习惯、财务管理、社交人脉、收获感悟。
- 目标进度变化时，继续向存在进度字段的关联模块同步进度（学习成长 `progress`、财务管理 `completion`）。
- 关键结果切换为真正的递归结构：前端表单与展示支持任意深度，后端 `toggle-kr` 使用路径数组定位节点，localStorage fallback 同步改造。
- 切换关键结果完成状态时，若一个父节点的所有子节点均已完成，则该父节点自动完成，并继续向上递归传播。

## Impact
- Affected specs: 目标管理、成就墙、关键结果数据模型、目标表单/详情、后端 goals API。
- Affected code:
  - `shared/schemas.ts`
  - `client/src/config/modules.ts`
  - `client/src/pages/GoalsPage.tsx`
  - `client/src/components/EntryForm.tsx`
  - `client/src/lib/api.ts`
  - `client/src/lib/storage.ts`
  - `server/src/routes/goals.ts`

## ADDED Requirements

### Requirement: 目标可关联成就
The system SHALL 在目标表单中展示所有手动成就（来自 `achievements` 表），按所属模块与父子层级分组；用户选择某个成就后，若该成就还有子成就，可继续选择下一级子成就；最终把选中的成就 ID 保存到 `linked_achievement_id`。

#### Scenario: 选择带子层级的成就
- **WHEN** 用户在「关联成就」中选择一个有子成就的父成就
- **THEN** 系统显示该父成就下的子成就供选择
- **WHEN** 用户保存目标
- **THEN** 目标记录中保存最终选中的子成就 ID

#### Scenario: 目标达成自动解锁成就
- **GIVEN** 一个目标已关联成就
- **WHEN** 该目标的关键结果全部完成（进度 100%）
- **THEN** 系统自动将关联成就的 `locked` 设为 `false`，`date` 设为今天
- **THEN** 前端刷新成就墙数据以反映解锁状态

### Requirement: 关键结果无限层级与父级自动完成
The system SHALL 支持关键结果任意层级的子节点；切换完成状态时，若一个父节点的所有子节点都已完成，则该父节点自动完成，并继续向上传播。

#### Scenario: 切换深层子节点
- **WHEN** 用户点击一个位于第三层以下的子关键结果的完成状态
- **THEN** 系统正确切换该子节点的完成状态

#### Scenario: 子节点全部完成导致父级自动完成
- **GIVEN** 某个父关键结果有两个子关键结果
- **WHEN** 用户将最后一个未完成的子关键结果标记为完成
- **THEN** 该父关键结果自动变为完成
- **WHEN** 该父级是另一个父级的子级，且其兄弟节点也都已完成
- **THEN** 上一级父关键结果也自动变为完成

#### Scenario: 父节点未全部完成时保持未完成
- **GIVEN** 某个父关键结果有两个子关键结果，只有一个完成
- **THEN** 该父关键结果保持未完成

## MODIFIED Requirements

### Requirement: 创建目标后自动在关联模块创建记录
The system SHALL 在创建目标时，如果用户选择了「关联模块」和「关联分类」，自动在对应模块创建一条记录；创建后把该记录 ID 回写到目标的 `linked_id`。目标进度变化时，向存在进度字段的模块同步进度。

#### Scenario: 关联学习成长并同步进度
- **GIVEN** 用户新建目标并选择「学习成长」及分类
- **WHEN** 保存目标
- **THEN** 学习成长模块出现一条以目标标题为内容、进度为 0 的记录
- **WHEN** 目标关键结果进度变化
- **THEN** 学习成长记录的 `progress` 字段同步更新

## REMOVED Requirements
无。
