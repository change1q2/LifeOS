# LifeOS - 里程碑时间轴与侧边栏优化 PRD

## Overview
- **Summary**: 重构里程碑模块为时间轴模式，自动聚合所有板块的内容并按时间顺序展示；同时实现侧边栏导航项的自由拖拽排序功能。
- **Purpose**: 用户希望能够以时间轴方式回顾自己在各个板块的活动记录，点击时间轴条目可以跳转到对应内容；侧边栏导航项可以自由移动位置以适应个人习惯。
- **Target Users**: LifeOS 系统的所有用户。

## Goals
- 将里程碑页面改为时间轴模式，自动获取所有板块的内容按时间顺序显示
- 时间轴条目点击后能够跳转到对应模块的详细内容
- 实现侧边栏导航项的拖拽排序功能
- 将里程碑导航项移到侧边栏最底部

## Non-Goals (Out of Scope)
- 不修改现有模块的数据结构和 API
- 不添加新的数据存储功能
- 不修改其他页面的布局和样式

## Background & Context
- 当前里程碑页面仅显示里程碑自己的数据，用户希望看到所有模块的时间线
- 侧边栏导航项当前是硬编码的固定顺序，用户无法自定义
- 需要从多个模块（学习、旅行、心情、目标、健康、财务、社交、感悟、成就、里程碑）中提取时间相关数据

## Functional Requirements
- **FR-1**: 时间轴模式自动聚合所有模块的内容
- **FR-2**: 时间轴条目按时间倒序排列，显示日期、模块图标、标题和摘要
- **FR-3**: 点击时间轴条目跳转到对应模块的详细页面
- **FR-4**: 侧边栏导航项支持拖拽排序
- **FR-5**: 拖拽排序结果持久化到 localStorage
- **FR-6**: 里程碑导航项移到侧边栏最底部

## Non-Functional Requirements
- **NFR-1**: 时间轴数据加载性能良好，支持并发请求多个模块
- **NFR-2**: 拖拽操作流畅，有视觉反馈
- **NFR-3**: 排序结果刷新页面后仍然保持

## Constraints
- **Technical**: React + TypeScript + Tailwind CSS，使用 @dnd-kit 或原生拖拽 API
- **Dependencies**: 需要考虑各模块数据结构的差异，提取统一的时间字段

## Assumptions
- 各模块都有可用于时间轴的日期字段（如 date、start_date、created_at 等）
- 用户浏览器支持 localStorage
- 用户希望按时间倒序查看所有活动

## Acceptance Criteria

### AC-1: 时间轴页面展示所有模块内容
- **Given**: 用户进入里程碑页面
- **When**: 页面加载完成
- **Then**: 页面显示时间轴模式，包含来自学习、旅行、心情、目标、健康、财务、社交、感悟、成就、里程碑等模块的内容，按时间倒序排列
- **Verification**: `human-judgment`

### AC-2: 时间轴条目显示日期和模块标识
- **Given**: 时间轴页面已加载
- **When**: 查看时间轴条目
- **Then**: 每个条目显示日期、模块图标、模块名称、标题和简短摘要
- **Verification**: `human-judgment`

### AC-3: 点击时间轴条目跳转
- **Given**: 时间轴页面已加载，有可点击的条目
- **When**: 点击时间轴条目
- **Then**: 跳转到对应模块的页面，并定位到该条目的详情
- **Verification**: `human-judgment`

### AC-4: 侧边栏导航项可拖拽排序
- **Given**: 用户查看侧边栏
- **When**: 用户拖拽导航项到新位置
- **Then**: 导航项移动到新位置，有视觉反馈（如阴影、占位符）
- **Verification**: `human-judgment`

### AC-5: 排序结果持久化
- **Given**: 用户完成拖拽排序
- **When**: 刷新页面
- **Then**: 侧边栏导航项保持用户自定义的顺序
- **Verification**: `programmatic`

### AC-6: 里程碑在侧边栏最底部
- **Given**: 页面加载完成
- **When**: 查看侧边栏导航项顺序
- **Then**: 里程碑导航项位于侧边栏导航列表的最底部
- **Verification**: `human-judgment`
